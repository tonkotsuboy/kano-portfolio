テーマのちらつき（FOUC）解消

必須CSS / JS

テーマ適用が useEffect 内（＝描画後）のため、ダーク利用者はページを開くたび一瞬ライトが見えます（suppressHydrationWarning がその傍証）。

解決策は <head> に ブロッキングな inline script を置き、描画前に localStorage を読んで data-theme を確定させること。現行の light/dark/system 3 択は維持できます。

Baseline localStorage / matchMedia は Widely available
対象 layout.tsx / ThemeProvider.tsx
根拠 dark-mode

skip-link を確実に機能させる

バグ修正HTML / a11y

① 記事詳細ページの <main> に id="main-content" が無く、このページだけ skip-link が機能しません（リンク先が存在しない）。

② 全ページの <main> に tabindex="-1" が無く、スキップ後にフォーカスが移らない場合があります。a11y ガイドは「スキップ先を focusable にする（<main id tabindex="-1">）」を明示。

影響 キーボード / スクリーンリーダー利用者
対象 記事詳細ページ ＋ 各 main
根拠 accessibility

LCP 画像の優先度指定

推奨Performance

記事詳細のカバー画像は priority 指定なし。記事詳細の LCP はこの画像なので付ける価値あり。トップは直近登壇カードの先頭サムネが候補。

どれが実際の LCP かは計測してから、最大 1〜2 枚に絞るのが鉄則（付けすぎは逆効果）。

Baseline Fetch Priority: Newly available（2024-10）

推奨CSS

Baseline Newly available（2024-07）/ 全主要対応

avatar の View Transition 引き継ぎ

CSS / JS

意図は「ページ間で avatar を滑らかに繋ぐ」。だが現状動いていない疑いが濃厚。

理由: ① <meta name="view-transition"> は出荷仕様に無く（正は CSS @view-transition{navigation:auto}）no-op の可能性。② cross-document VT は実ドキュメント遷移時のみ発火 → Next.js の <Link>（same-document）では発火しない。③ same-document VT は React <ViewTransition>（canary）依存。→ no-op な宣言／コメントの整理を推奨。

React <ViewTransition>（same-document）

JS

クライアント遷移にアニメを付ける本命だが安定版に未到来。

理由: 一度導入して revert 済み。React canary 依存を個人サイトの本番に持ち込むのはリスク。stable 化を待つ。

content-visibility: auto

CSS

オフスクリーン要素の描画を遅延してパフォーマンス改善。

理由: ArticleGrid（flex）と UpcomingTalks（subgrid）の両方で高さ計算が崩れる現象を確認し撤去済み（コード内コメントに明記）。採用しない方針が確定済み。

scheduler.postTask() / yield()

JS

長いタスクを分割して INP を守る API。

理由: Safari 未対応（要 polyfill）。加えて Client JS が薄く、分割対象の重い処理が無い。恩恵が費用に見合わない。

スクロールでヘッダー縮小

CSS

スクロール量でヘッダーを小さくする演出。

理由: height アニメは合成スレッドに乗らずカクつきやすい（ガイドも警告）。ヘッダーは既にコンパクト（64px）で余地が小さく、「静かで丁寧」な方針にも不一致。

カードのスクロールイン演出

CSS

カードがビューに入るとフェード／スケールイン（animation-timeline: view()）。

理由: 技術的には可能だが、ミニマル志向に対し装飾過多。View Transitions を revert した精神と同じく見送り。
